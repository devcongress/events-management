## Quick Summary

**DevCon-Comm** — Vue 3 + Vite community tech conference platform for DevCongress, served through a single Bun/Hono app server. Manages the full talk lifecycle (CFP → review → slides → archive) and runs a Kahoot-style live quiz. Prototype data still uses JSON flat files in several areas; hosted admin auth uses Supabase Google OAuth with app-owned HTTP-only sessions.

**Stack:** Vue 3 · Vite 6 · TypeScript 5 · Bun · Hono · Pinia · Tailwind CSS 3 · pnpm

## Key Facts (Instant Orientation)

- **App shell:** `src/` Vue SPA with Vue Router; `server/` Hono API and Bun production server
- **Legacy reference:** previous Next.js implementation remains in `app/`, `components/`, and `hooks/` during migration
- **Admin auth** — hosted organizer routes use Supabase Google OAuth plus app-owned HTTP-only sessions; local dev can fall back to `ADMIN_PASSWORD`
- **Quiz real-time** — polling every 1500ms; clients call explicit `POST /api/quiz/state/advance` before read-only state fetches
- **Scoring** — `base × (0.5 + 0.5 × speed)` + streak bonus (2→+100 … 5+→+500)
- **Mock DB** — `lib/mock-db/index.ts`: `readData<T>` / `writeData<T>` over `data/*.json`, with corrupt JSON failing loudly and writes using temp-file rename
- **Design tokens** — `dc-*` Tailwind classes + `lib/design-system.ts` for JS-side usage
- **Seed DB** — `pnpm seed`
