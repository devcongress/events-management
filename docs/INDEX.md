## Quick Summary

**DevCon-Comm** — Vue 3 + Vite community tech conference platform for DevCongress, served through a single Bun/Hono app server. Manages the full talk lifecycle (CFP → review → slides → archive) and runs a Kahoot-style live quiz. Prototype stage: JSON flat-file mock DB with same-origin prototype admin cookie auth.

**Stack:** Vue 3 · Vite 6 · TypeScript 5 · Bun · Hono · Pinia · Tailwind CSS 3 · pnpm

## Key Facts (Instant Orientation)

- **App shell:** `src/` Vue SPA with Vue Router; `server/` Hono API and Bun production server
- **Legacy reference:** previous Next.js implementation remains in `app/`, `components/`, and `hooks/` during migration
- **Prototype admin auth** — `/admin/*` routes use a same-origin cookie session; default local password is `devcon-admin` unless `ADMIN_PASSWORD` is set
- **Quiz real-time** — polling every 1500ms; server auto-advances phases inside GET handler
- **Scoring** — `base × (0.5 + 0.5 × speed)` + streak bonus (2→+100 … 5+→+500)
- **Mock DB** — `lib/mock-db/index.ts`: `readData<T>` / `writeData<T>` over `data/*.json`
- **Design tokens** — `dc-*` Tailwind classes + `lib/design-system.ts` for JS-side usage
- **Seed DB** — `pnpm seed`
